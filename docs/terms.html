Last updated: 2025-09-16

Purpose

This application is a personal, non-commercial demonstration for accessing your own health records from Epic/MyChart via SMART on FHIR (FHIR R4). It is intended for testing and educational use only.

Data Access and Storage

  • The app requests read-only access to your data using the scopes: openid, fhirUser, launch/patient, patient/*.read, and offline_access.
  • Access and refresh tokens are stored only on your local machine in a file named tokens.json. No data or tokens are uploaded to any server controlled by the app author.
  • FHIR data is fetched directly from the Epic FHIR server and displayed to you in the app. The app does not maintain a backend database.
  • To remove local data, delete the tokens.json file in the app folder.

Revoking Access

You can revoke this app's access at any time from your MyChart account settings (look for Active App Connections / Third-Party Apps) and remove the local tokens.json file.

Privacy

  • No analytics, tracking pixels, or third-party ad services are used.
  • Logs are kept only on your local machine during app execution.

Security

  • For OAuth redirect, the app supports HTTPS on https://127.0.0.1:8765/callback. If using a self-signed certificate locally, your browser may prompt you to proceed.
  • You are responsible for securing your local machine where tokens are stored.
